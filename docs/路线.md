# S0 — 剧情/故事系统（优先实现）

> 要实现火焰纹章式的主线剧情，需要三个底层基础设施：
> 对话框系统 + 剧情编辑器/加载器 + 存档系统。
> 这些做好之后，写剧情就只是写 JSON 的事了。

## S0.1 对话框系统

**概述**：一个通用、可复用的对话框组件，用于剧情对话、角色说话、系统提示。

**技术要求**：
- 纯 CSS 底部弹出面板（类似火纹/AVG 对话框）
- 支持：角色名 + 角色头像占位 + 对话文本 + 逐字显示（打字机效果）
- 支持：点击任意处快速显示全文 / 点击下一步
- 支持：选项分支（最多 4 个选项，选后跳转到不同对话段落）
- 支持：动画进出（底部滑入/淡入）

**改动范围**：
- `web/index.html`：对话框容器
- `web/style.css`：对话框样式（底栏、头像框、文字区、选项按钮）
- `web/app.js`：`dialog.show(texts, opts)` 函数

**状态**：[✓] （`Dialog` 模块已实装于 `app/web/app.js`：底部对话框 + 全屏叙述；支持 dialogue/narration/choice/wait 四种 scene；打字机 40 字/秒、叙述 25 字/秒；点击/空格/Enter 推进、Esc 关闭；31/31 jsdom 单测通过）

### ⏸ 需要确认的技术细节

## S0.2 剧情系统

**概述**：定义一套 JSON 剧情格式 + 加载器，剧情脚本驱动对话框。

**JSON 剧情格式草案**：

```json
{
  "id": "chapter_01_intro",
  "scenes": [
    {
      "type": "dialogue",
      "speaker": "云",
      "speaker_color": "#ffb84d",
      "text": "这片大陆已经被黑暗笼罩了太久……",
      "next": "scene_2"
    },
    {
      "type": "dialogue",
      "speaker": "红",
      "speaker_color": "#e85a6a",
      "text": "但今天，我们将吹响反击的号角。",
      "next": "scene_3"
    },
    {
      "type": "choice",
      "question": "你选择相信谁？",
      "choices": [
        {"text": "相信云的判断", "next": "scene_4a"},
        {"text": "跟随红的勇气", "next": "scene_4b"}
      ]
    },
    {
      "type": "dialogue",
      "id": "scene_4a",
      "speaker": "云",
      "text": "你的信任不会白费。",
      "next": "scene_5"
    },
    {
      "id": "scene_4b",
      "speaker": "红",
      "text": "这才是我认识的你！",
      "next": "scene_5"
    },
    {
      "type": "battle",
      "id": "scene_5",
      "map_preset": "mountain_pass",
      "unit_composition": "classic",
      "win_condition": "rout",
      "next_story": "chapter_01_after"
    }
  ]
}
```

**剧情格式支持的类型**：
| type | 效果 |
|------|------|
| `dialogue` | 对话框显示文本，支持角色名+颜色 |
| `choice` | 显示选项分支，不同选项不同下一场景 |
| `battle` | 过渡到战斗（指定地图/兵种/胜利条件）|
| `narration` | 全屏文字叙述（无对话框边框）|
| `wait` | 等待 N 秒后自动继续（用于演出效果）|
| `condition` | 根据玩家存档数据判断分支（如"第 2 章时士气＞3"）|

**游戏流程**：游戏大厅 → 选择"剧情模式" → 按章节播放剧情 → 剧情末尾进入战斗 → 战斗胜利 → 继续剧情 → ...

**改动范围**：
- `app/game_logic.py`：剧情 JSON 加载器
- `app/routes/game.py`：`POST /games/story/{chapter_id}` 端点
- `web/app.js`：`StoryEngine` 类——驱动场景播放
- `web/index.html`：剧情模式入口按钮
- `game/stories/`：剧情 JSON 文件夹

**状态**：[✓] （4 步全完成，2026-06-27）

## S0.3 存档系统

**概述**：玩家可以随时保存/读取/删除存档。每个存档记录该玩家的游戏进度。

**数据模型**：

```json
{
  "id": "save_1",
  "player_name": "玩家",
  "chapter_id": "chapter_01",
  "scene_id": "scene_5",
  "game_state_snapshot": {
    "turn": 3,
    "players": [{...}],
    "units": [{...}],
    "tiles": [{...}]
  },
  "player_data": {
    "gold": 1500,
    "units_total_kills": 47,
    "total_turns_played": 12,
    "morale_accumulated": 9
  },
  "created_at": "2026-06-25T12:00:00Z"
}
```

**改动范围**：
- `models.py`：`SaveSlot` ORM 模型（id, player_name, chapter_id, scene_id, snapshot:JSON, player_data:JSON, created_at）
- `schemas.py`：保存/读取/删除 requests + responses
- `routes/game.py`：
  - `POST /games/save` — 创建存档
  - `GET /games/saves` — 列出该玩家所有存档
  - `POST /games/saves/{id}/load` — 读取存档
  - `DELETE /games/saves/{id}` — 删除存档
- `web/app.js`：
  - 保存按钮（游戏内）
  - 读档/存档列表界面
  - 自动存档（每回合开始自动存）

**状态**：[✓]（主线存档子集已实装：见 [mainline/steps.md](mainline/steps.md) Step 2，`PlayerProfile.mainline_progress` 字段按 `user_name` 持久化主线战斗序号 + scene_id；23/23 单测通过）。自由模式通用存档 SaveSlot 暂未实装，待 P0+ 平衡后再补。

### ⏸ 需要确认的技术细节

1. 存档是绑定玩家名还是需要未来加登录？
   - 推荐：先绑定玩家名（读取时输入名字），未来再加 token 认证
2. 存档数量限制？（推荐：每个名字最多 10 个存档位 + 1 个自动存档）
3. 存档是否包含全部 game_state 快照还是只存关键数据？
   - 推荐：全部快照（省去重建逻辑），JSON 大小约 20-50KB
4. 是否支持"战斗中断存"？（战斗中随时可以存，读档后回到存的那一刻）→ 推荐：支持
