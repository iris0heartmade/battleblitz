## 🐛 Bugs

### 1. 击杀奖励会始终累积在一个棋子上  —  [—] 非 bug（设计如此）
- **结论**：不是 bug。`actions.py:335-337` 中 `is_kill → award_morale(attacker)` 与 `award_exp(attacker, "kill")` 都正确指向攻击者；`game_logic.py:346-360` `award_morale` 直接在 `attacker` 对象上加 morale，**累加是正确的**。若想"分散给全队"，是设计变更不是 bug 修复。
- **建议**：维持现状。如要改，明确说明是设计变更。

### 2. 被击杀的棋子不会马上消失  —  [ ] 部分修复，仍存在
- **现状**：击杀判定 `is_kill = target.hp <= 0` 在 `actions.py:332` 立即生效，但 `cleanup_dead_units` 仅在 `apply_end_of_turn`（`game_logic.py:387-401`）时执行。前端 `app.js:511-512` 用 `hp > 0` 过滤，所以**下一轮 `refreshGame()` 轮询**才能让死单位消失（典型 1-3 秒延迟）。
- **修复方案**：在 `actions.py:attack` 端点结算后立即调 `cleanup_dead_units`（不仅 end-of-turn）。改动小，可立即修复。

### 3. 手机地图方格空隙/无法完整显示  —  [ ] 部分修复，仍存在
- **现状**：`style.css:1301-1332` 有两档断点（768px→28px / 480px→22px），但用缩小格子尺寸代替连续缩放，无 `transform: scale` 或 viewport units 适配。在 320-767px 之间仍可能溢出。
- **修复方案**：加 JS 动态计算 `boardTransform = min(vw/boardWidth, vh/boardHeight)` 后 `board.style.transform = scale(...)`，或 CSS `transform-origin: top left` + 容器 overflow 隐藏。
